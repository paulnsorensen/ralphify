Put your AI coding agent in a while True loop and let it ship.

Ralphify is a minimal CLI harness for autonomous AI coding loops. Pipe a prompt to an agent, validate with checks, repeat.

How it works

Every iteration follows the same cycle. The agent always starts fresh — no conversation history, no stale context.

1

Read prompt + contexts

Ralphify reads your RALPH.md prompt, runs context commands (git log, test status, etc.), and injects the results into the prompt via placeholders.

→
2

Agent does one thing

The assembled prompt is piped to your agent. It reads the codebase, implements one task, runs tests, and commits. Then it exits.

→
3

Checks validate & feed back

Checks (tests, linters) run after the agent exits. If something fails, the output feeds into the next iteration — the agent fixes its own mistakes.

Then it repeats. Edit the prompt while it runs — changes take effect on the next cycle.

Why ralph loops?

A single agent conversation fills up its context window, slows down, and loses the plot. Ralph loops solve this by resetting every iteration.

↻

Fresh context, no decay

Each iteration starts with a clean context window. No conversation bloat, no hallucinated memories. The agent reads the current codebase every loop.

⚔

Self-healing feedback

Checks validate the agent's work after each iteration. When something breaks, the failure feeds into the next iteration automatically — the agent fixes its own mistakes.

✎

Steer while it runs

The prompt is re-read every iteration. Edit it while the loop runs and the agent follows your new rules on the next cycle. When it does something dumb, add a sign.

⎇

Progress lives in git

Every iteration commits to git. If something goes wrong, git log shows exactly what happened and git reset rolls it back. No opaque state to debug.

Install

uv tool install ralphify

Three primitives

Extend the basic loop with building blocks that live in .ralphify/

Checks

Run after each iteration to validate the agent's work. Failed check output feeds into the next iteration so the agent can fix its own mistakes.

Contexts

Inject dynamic data into the prompt before each iteration — git history, test status, API responses. The agent always sees fresh information.

Ralphs

Named, task-focused prompts you switch between at run time. Keep a docs ralph, a refactor ralph, and a bug-fix ralph.

Two commands to start

ralph init && ralph run
