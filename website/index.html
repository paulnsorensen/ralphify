The primitive

A ralph is just
a Markdown file

No config files. No project scaffolding. No dashboard.
One file is the whole thing.

  • agent — which CLI agent to run
  • commands — checks that validate each iteration
  • The prompt body — what the agent sees every loop
RALPH.md 
---
agent: claude-code
commands:
  - name: tests
    run: uv run pytest --tb=short
---

You are an autonomous coding agent running in a loop.

Test results from the last run:
{{ commands.tests }}

Fix any failing tests. If all tests pass,
pick the next task from TODO.md and implement it.

How it works

The loop

Three steps, every iteration. Fresh context, no drift.

1

Assemble the prompt

Ralphify reads your RALPH.md, runs commands to gather live data, and injects results into the prompt.

2

Pipe to the agent

The assembled prompt goes to your agent. It reads the codebase, does one unit of work, commits, and exits.

3

Repeat

Back to step 1. Fresh context window, latest code. Edit the prompt while it runs — changes take effect next cycle.

↩ back to step 1

Why loops

Single runs decay.
Loops compound.

A single agent conversation fills up, slows down, and loses the plot. Ralph doesn't.

>_

Fresh context, no decay

Each iteration starts with a clean context window. No conversation bloat. The agent reads the actual codebase every loop.

</>

Self-healing feedback

When something breaks, the failure output feeds into the next iteration automatically. The agent sees what went wrong and fixes it.

~/

Steer while it runs

The prompt is re-read every iteration. Edit it while the loop runs and the agent follows your new direction next cycle.

∞

Progress lives in git

Every iteration commits. If something goes sideways, git log shows exactly what happened.

FAQ

What AI coding agents work with Ralphify?
Any agent that accepts prompts via stdin or CLI arguments — Claude Code, Aider, Codex CLI, and others. Ralphify is agent-agnostic by design. See supported agents.
What is a ralph loop?
An autonomous AI coding cycle: assemble a prompt, pipe it to an agent, let the agent work, repeat. Each iteration starts fresh — no stale context, no accumulated conversation history.
How is this different from running my agent once?
A single agent run fills up its context window, slows down, and loses focus. Ralphify resets every iteration — fresh codebase, fresh data, fresh prompt. You steer, the loop executes.
Does Ralphify need a web UI?
No. It's a pure CLI tool. Runs in your terminal, works over SSH, no browser needed.
